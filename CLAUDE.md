# Правила работы с проектом — Stas breathwork app

> Этот файл автоматически читается Claude Code при каждом запуске.

## Git: безопасность изменений

- git stash перед любыми изменениями — убедись что working tree чистый
- НИКОГДА не смешивай несвязанные изменения в одном коммите
- После завершения работы: `npm run build` → исправь ВСЕ ошибки → только тогда commit и push

## Перед каждой правкой

1. `git stash` или `git commit -am "checkpoint"` — точка отката
2. `npm run build` — убедись что база чистая
3. Проверь что working tree чистый: `git status`

## Режим работы: сначала план, потом код

- ВСЕГДА сначала покажи план БЕЗ кода:
  - Какие файлы менять (список)
  - Что именно в каждом файле (кратко)
  - Риски регрессий
  - Чек-лист ручной проверки (3-5 шагов)
- Дождись подтверждения "ок" от пользователя
- Только потом пиши код

## Правила изменений

- Маленький дифф: максимум 3 файла за раз (если больше — разбей на шаги)
- Не делай рефакторинг без явного запроса
- Не меняй форматирование вне затронутых строк
- Не переименовывай файлы/функции без причины
- Не добавляй зависимости без согласования
- Не удаляй "мёртвый код" без явного запроса — он может использоваться

## После каждой правки

1. `npm run build` — убедись что билд проходит
2. Исправь ВСЕ ошибки если есть
3. `git diff` — покажи что изменилось
4. Дай чек-лист ручной проверки (3-5 шагов)
5. Только после успешного билда — `git add . && git commit && git push`

## При баге — сначала диагностика

- НЕ предлагай фикс сразу
- Сначала:
  - 3 вероятные причины
  - Где в коде искать (файлы + функции)
  - Какие console.log / логи добавить для диагностики
  - Как воспроизвести стабильно
- Потом — самый маленький фикс

## Архитектура проекта

- **Стек:** Vite + React 18, деплой на Cloudflare Pages
- **UI логика** — только в `src/components/`
- **Данные (контент практик)** — в `src/data/practices.js`
- **Внешние SDK (Telegram WebApp)** — только через `src/utils/telegram.js`, НИКОГДА в компонентах
- **Стили** — inline styles + глобальные анимации в `src/styles.css`
- **Аудио** (когда появится) — файлы в `public/audio/`, URL в `practices.js`
- Никаких внешних UI-библиотек — всё кастомное

## Структура файлов

```
src/
├── App.jsx                    # Главный экран
├── main.jsx                   # Entry point
├── styles.css                 # Глобальные стили и анимации
├── components/
│   ├── GenArt.jsx             # Генеративный арт (canvas)
│   ├── IntensityDots.jsx      # Индикатор интенсивности
│   ├── PracticeCard.jsx       # Карточка практики
│   ├── PracticeDetail.jsx     # Детальный экран (bottom sheet)
│   └── BreathPlayer.jsx       # Полноэкранный плеер
├── data/
│   └── practices.js           # Контент: практики, категории
└── utils/
    └── telegram.js            # Telegram WebApp API
```

- Менять структуру папок
